# 日志文件说明

## 日志文件位置

打包后运行时，日志文件在**用户选择的数据目录**下的 `logs/` 子目录：

```
C:\Users\用户名\选择的目录\
├── logs\
│   ├── app.log        ← Spring Boot 应用日志（启动诊断、保存操作等）
│   └── jvm.log        ← JVM 控制台输出（启动错误、崩溃信息等）
├── config.json
├── 生成结果\
├── .temp-output\
└── ...
```

## 两个日志文件的作用

### 1. `app.log` - 应用日志
- **内容**：Spring Boot 应用的所有日志（INFO 级别）
- **配置**：`logback-spring.xml`（已打包进 jar）
- **滚动策略**：单文件 10MB，按天滚动，保留 7 天，总上限 50MB

**关键记录**：
- ✅ **启动诊断**（每次启动自动记录）：
  ```
  ========== 启动环境诊断 ==========
  user.dir (工作目录) = C:\Users\用户\数据目录
  user.home (用户目录) = C:\Users\用户
  app.packaged (打包标记) = true
  [.temp-output (临时归档)] 路径=... 已存在，可写=true
  [生成结果 (永久输出)] 路径=... 已存在，可写=true
  ```

- ✅ **保存操作详细日志**：
  ```
  save-to-gallery 请求: tempPath=..., subDir=...
  路径校验 - tempPath原始: C:\Users\...\1.jpg
  路径校验 - tempRoot: C:\Users\...\\.temp-output
  路径校验 - source标准化: C:\Users\...\1.jpg
  路径校验 - source.startsWith(tempRoot): false  ← 如果是 false 就是路径问题
  ```

- ✅ **错误信息**：
  ```
  ERROR ResourceController - 路径校验失败 - source: ..., tempRoot: ...
  WARN  ResourceController - 源文件不存在: ...
  ERROR ResourceController - 无法创建目标目录: ...
  ```

### 2. `jvm.log` - JVM 控制台输出
- **内容**：Java 进程的 stdout/stderr（JVM 启动错误、崩溃堆栈等）
- **配置**：Electron main.js 捕获并写入
- **用途**：诊断 Java 进程启动失败、内存溢出、严重崩溃等底层问题

---

## 如何获取日志用于诊断

### 方法 1：让用户发送日志文件

1. 在应用内打开帮助菜单（或发给用户这个说明）
2. 让用户打开数据目录：
   - 启动时用户选择的那个目录
   - 或者在设置界面显示当前数据目录路径
3. 找到 `logs/` 文件夹
4. 把 `app.log` 和 `jvm.log` 发给你

### 方法 2：在应用内添加"导出日志"按钮（可选）

前端添加一个按钮，调用后端接口打包 `logs/` 目录为 zip，用户直接下载。

## 典型问题诊断

### 问题 1：右键保存失败

**日志特征**：
```
save-to-gallery 请求: tempPath=C:\Users\张三\数据目录\.temp-output\xxx\1.jpg
路径校验 - source.startsWith(tempRoot): false
ERROR 路径校验失败
```

**原因**：路径格式不匹配或路径校验逻辑过严

**解决方案**：放宽路径校验（见 `问题排查指南.md` 方案 1）

---

### 问题 2：临时文件已被清理

**日志特征**：
```
save-to-gallery 请求: tempPath=...
WARN 源文件不存在: C:\Users\张三\数据目录\.temp-output\xxx\1.jpg
```

**原因**：用户超过 2 小时才点保存，临时文件已被 TTL 清理

**解决方案**：
1. 延长 TTL 到 24 小时
2. 前端提示用户尽快保存

---

### 问题 3：目录无写权限

**日志特征**：
```
[.temp-output (临时归档)] 路径=... 不存在，尝试创建：失败（可能无写权限）
或
ERROR 无法创建目标目录: C:\Users\张三\数据目录\生成结果\20260621
```

**原因**：用户选择的数据目录在受限位置（如 C:\Program Files）

**解决方案**：
1. 让用户重新选择数据目录（有写权限的位置）
2. Electron 启动时检测目录可写性，不可写时提示重新选择

---

### 问题 4：Java 进程启动失败

**日志特征**：`jvm.log` 中有错误信息，但 `app.log` 为空或不存在

**可能原因**：
- JRE 版本不匹配
- 端口 5020 被占用
- jar 文件损坏

**查看** `jvm.log` 获取具体错误

---

## 开发环境下的日志

开发环境（直接运行 `mvn spring-boot:run`）：
- **位置**：项目根目录的 `logs/app.log`
- **user.dir**：`F:\java\ele-business-java`
- **app.packaged**：false

---

## 注意事项

1. **日志文件会自动滚动**：不用担心占满磁盘（总上限 50MB）
2. **敏感信息**：日志中不记录 API Key（已在代码中屏蔽）
3. **查找日志**：如果用户找不到数据目录，让他们在应用设置界面查看当前路径
