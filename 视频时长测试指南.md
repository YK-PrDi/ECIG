# 视频时长测试指南

## 目的
测试各视频提供商支持的时长范围（1s ~ 15s），验证实际生成效果。

## 测试方法

### 前端界面测试
1. 登录系统：http://localhost:5020
2. 进入"视频模式"
3. 选择要测试的模型
4. 填写简单提示词（如："花洒出水特写"）
5. 在"时长（秒）"下拉框选择不同时长：1s, 2s, 3s...15s
6. 点击"开始生成视频"
7. 记录生成结果

### API直接测试
```bash
# 测试Grok文生视频 - 1秒
curl -X POST "http://localhost:5020/api/video/generate" \
  -H "Cookie: JSESSIONID=xxx" \
  -F "model=grok-imagine-video" \
  -F "prompt=花洒出水特写" \
  -F "aspectRatio=16:9" \
  -F "durationSeconds=1" \
  -F "sessionId=test"

# 测试即梦 - 15秒
curl -X POST "http://localhost:5020/api/video/generate" \
  -H "Cookie: JSESSIONID=xxx" \
  -F "model=as-sd2.0-fast" \
  -F "prompt=花洒出水特写" \
  -F "aspectRatio=16:9" \
  -F "durationSeconds=15" \
  -F "sessionId=test"
```

## 测试注意事项

1. **成本控制**
   - 每次测试消耗积分/余额
   - 建议从中间值（5-8s）开始测试
   - 确认可用后再测试边界值（1s和15s）

2. **错误处理**
   - 记录API返回的具体错误信息
   - 如报"参数错误"，说明该时长不支持
   - 如报"欠费"，需充值后再测试

3. **质量评估**
   - 1-3s: 极短视频，可能只是简单动画
   - 4-8s: 标准短视频，内容完整
   - 9-15s: 长视频，内容丰富但成本更高

4. **提示词建议**
   - 简单场景："花洒出水特写"
   - 避免复杂场景，减少生成失败风险
   - 保持提示词一致，便于对比不同时长

## 代码限制说明

### 即梦时长限制（已实现）
```java
// OpenAiCompatibleVideoService.java submitJimeng方法
.addFormDataPart("seconds", String.valueOf(Math.max(1, Math.min(15, durationSeconds))))
```
- 前端可选1-15s
- 后端自动限制在1-15s范围内
- 小于1取1，大于15取15

### 其他提供商
- 目前无后端时长限制
- 由各提供商API自行验证
- 如API报错，需记录并更新此文档
