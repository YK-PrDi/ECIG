# 仓库级工作规则

## 强制验收门禁

凡修改业务代码、前端资源、配置、构建产物或部署内容，必须先阅读并执行：

- [`docs/运行与发布验收门禁.md`](docs/运行与发布验收门禁.md)

只有该文档中与本次变更相关的强制项全部执行成功，并保留可复核的命令结果后，才可以使用“运行正常”“修复完成”“部署成功”或同等表述。

以下情况一律不能判定为正常：

- 只完成代码修改，没有运行测试。
- 只运行局部测试，没有运行全量测试。
- 本地通过，但生产环境未验证。
- 测试因账号、网络、依赖或权限无法执行。
- 部署后没有检查服务状态、HTTP、日志和文件哈希。
- 浏览器关键流程没有执行对应的自动化回归检查。

无法执行的项目必须明确标记为“阻塞”或“未验证”，说明原因、影响和下一步，不得把“理论上可用”写成“已正常运行”。

## 基本要求

- 修复缺陷时先增加能复现问题的失败测试，再实施修复。
- 不提交、输出或上传 `.env.local`、服务器环境变量、API Key、密码和令牌。
- 部署前必须备份；部署失败必须回滚并重新验证。
- 不覆盖或回退工作区中与当前任务无关的现有修改。
