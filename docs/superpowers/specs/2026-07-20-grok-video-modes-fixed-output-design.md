# Grok 视频模式拆分与固定输出设计

## 目标

将 Grok 视频能力拆分为两个独立下拉项，并保证用户选择的视频比例最终对应固定 MP4 尺寸。

## 模型契约

- `grok-imagine-video` 显示为“Grok 文生视频”，只接受提示词，不允许参考图。
- `grok-imagine-video-1.5` 显示为“Grok 图生视频”，必须且只能使用一张参考图。
- 两个模型均调用随想兼容端点 `/v1/videos/generations`。
- 文生请求不包含图片字段；图生请求使用随想兼容字段 `image_url`。
- 前端负责即时提示，后端负责最终校验，非法输入不得请求供应商。

## 前端交互

- 视频模型下拉框直接展示“Grok 文生视频”和“Grok 图生视频”。
- 两个 Grok 选项各自只有一个模型，因此不展示模型等级滑条。
- 即梦仍作为一个供应商分组，保留其两档模型滑条。
- 文生视频检测到参考图时阻止提交；图生视频未上传或上传多张参考图时阻止提交。

## 固定输出

视频下载完成后由服务端 FFmpeg 规范输出：

| 比例 | 固定尺寸 |
|---|---|
| 16:9 | 1280x720 |
| 9:16 | 720x1280 |
| 1:1 | 720x720 |
| 4:3 | 960x720 |
| 3:4 | 720x960 |
| 3:2 | 1080x720 |
| 2:3 | 720x1080 |

统一输出为 MP4、H.264、`yuv420p`、方形像素，并启用 faststart。尺寸不匹配时采用等比放大后中心裁剪；尺寸匹配时执行轻量重封装，避免无必要的画质损失。

## 错误处理

- FFmpeg 或 FFprobe 不可用时任务失败并返回明确错误，不把未经验证的文件标记为成功。
- 转码失败时删除临时文件，保留原任务错误信息，不写成功历史。
- 取消任务时终止正在运行的 FFmpeg/FFprobe 子进程。

## 验收

- 自动测试覆盖模型分组、文生/图生请求字段、输入校验和尺寸映射。
- 前端检查覆盖两个 Grok 下拉项和 Grok 滑条隐藏行为。
- Maven 全量测试、前端门禁、构建和生产部署检查全部通过。
- 仅执行一次真实 `grok-imagine-video-1.5`、一张参考图、`3:4`、`4s` 调用，最终文件必须为 `720x960`。
