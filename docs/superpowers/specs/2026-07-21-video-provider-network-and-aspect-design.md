# 视频供应商网络与尺寸一致性设计

## 目标

1. 让生产服务器能够稳定下载 Grok 返回的 `vidgen.x.ai` 视频文件。
2. 让即梦请求参数中的尺寸与系统最终输出尺寸保持一致。
3. 完成自动测试、真实供应商验证、打包、备份、部署和运行维护检查。

## 现状与根因

- 随想主 API 的提交和轮询可以从生产服务器正常访问。
- Grok 失败发生在媒体下载阶段，生产服务器直连 `vidgen.x.ai:443` 超时。
- 之前本地 Grok 成功时，Java 请求使用了本地显式代理；生产服务器目前没有代理出口。
- 即梦已经发送 `size` 字段，视频完成后也会经过 FFmpeg 规范化，但上游请求尺寸和最终输出尺寸各自维护了一份映射，存在后续不一致风险。

## 方案选择

采用媒体专用 WARP 代理，不改变服务器默认路由：

- 在生产服务器安装 Cloudflare WARP 官方 Linux 客户端。
- WARP 以本地代理模式运行，仅监听服务器回环地址的 SOCKS5 端口。
- Java 只对 Grok 返回的媒体下载域名使用该 SOCKS5 代理。
- 随想主 API、即梦、数据库、登录、静态资源和其他模型继续使用原有直连方式。

不采用全局 WARP 路由，避免改变 SSH、数据库和其他供应商的网络行为。不使用固定 CDN IP，避免 DNS 调度、TLS 证书和节点变更问题。

## 应用配置

在 `app.suixiang-video` 下增加媒体代理配置，并由生产环境变量注入：

```yaml
media-proxy:
  enabled: false
  type: socks5
  host: "127.0.0.1"
  port: 40000
  hosts:
    - "vidgen.x.ai"
```

生产环境变量：

```text
SUIXIANG_MEDIA_PROXY_ENABLED
SUIXIANG_MEDIA_PROXY_TYPE
SUIXIANG_MEDIA_PROXY_HOST
SUIXIANG_MEDIA_PROXY_PORT
SUIXIANG_MEDIA_PROXY_HOSTS
```

配置中不包含 API Key、代理订阅或其他敏感值。代理只绑定 `127.0.0.1`，不对公网开放。

## Grok 下载流程

1. 创建和轮询 Grok 任务仍使用随想主 API 的普通客户端。
2. 得到媒体 URL 后解析主机名。
3. 主机命中媒体代理白名单时，使用 SOCKS5 媒体客户端下载。
4. 同一个媒体 URL 最多尝试三次，使用短退避，不重新提交生成任务。
5. 每次失败删除不完整 `.part` 文件；任务取消时立即停止等待和网络调用。
6. 日志只记录媒体主机、尝试次数和异常类型，不记录 API Key、签名参数或完整下载 URL。
7. 三次下载仍失败时返回明确的“媒体下载网络不可达”错误，任务记为失败且不保存空文件。

## 尺寸统一

新增单一尺寸规格组件，作为所有视频比例映射的唯一来源：

| 比例 | 宽 | 高 | API size |
|---|---:|---:|---|
| 16:9 | 1280 | 720 | 1280x720 |
| 9:16 | 720 | 1280 | 720x1280 |
| 1:1 | 720 | 720 | 720x720 |
| 4:3 | 960 | 720 | 960x720 |
| 3:4 | 720 | 960 | 720x960 |
| 3:2 | 1080 | 720 | 1080x720 |
| 2:3 | 720 | 1080 | 720x1080 |

- 即梦 multipart 请求的 `size` 从该规格生成。
- FFmpeg 最终输出目标也从该规格生成。
- 非法比例统一回退到 `16:9`。
- Grok 仍发送供应商要求的比例字符串，并由最终规范化保证实际文件尺寸。

## 测试

自动测试必须覆盖：

- 媒体白名单命中时使用 SOCKS5，随想主 API 不使用媒体代理。
- 非白名单媒体 URL 保持直连。
- 同一 URL 下载失败后重试，成功后只产生一个完整文件。
- 取消任务时停止重试并删除部分文件。
- 七种比例的即梦 `size` 与最终 FFmpeg 目标完全一致。
- `3:4` 必须是 `720x960`，`16:9` 必须是 `1280x720`。
- 现有前端、认证、并发、取消和全量 Maven 门禁继续通过。

## 部署与维护

1. 安装 WARP 前记录服务器状态并备份服务配置。
2. WARP 代理模式必须由 `warp-svc` 管理并开机启动。
3. 先用 `curl --socks5-hostname` 验证 `vidgen.x.ai`，通过后才部署应用。
4. 备份当前 JAR、前端和 systemd 配置。
5. 上传临时文件、校验 SHA-256、原子替换并重启 `ai-studio`。
6. 失败时恢复应用备份；WARP 保持独立，不影响服务器默认路由。

## 生产验收

- Grok 图生视频：一张参考图、`3:4`、`4s`，任务成功并下载。
- 即梦：`3:4`、`4s`，任务成功并下载。
- 两个文件均用 FFprobe 验证为 MP4、H.264、`yuv420p`、`720x960`，时长约 4 秒。
- `ai-studio` 和 `warp-svc` 均为 active，应用重启次数为 0。
- 首页、登录、模型目录、任务轮询和视频下载接口通过。
- 部署后日志不存在 Bean、数据库、FFmpeg、代理泄漏或持续网络异常。
