# 羽刃创作工作台高级感与操作稳定性改造设计

## 1. 目标

在不推翻现有三栏工作台和后端生成接口的前提下，完成一次面向真实操作的前端精修：

- 保留顶部应用导航、左侧画布工具、中间无限画布、右侧生成对话区的总体框架。
- 模式切换继续位于生成对话区顶部，不移入全局应用导航。
- 采用“银灰专业创作台”视觉基调，通过留白、层级、尺寸和边界规则提升高级感。
- 消除标签挤压、面板边缘碰撞、下拉层被裁切、画布缩放后错位等常见 UI 小问题。
- 补全局部编辑的可见工具和提交状态，确保原图、蒙版和指令作为一个完整请求发送给模型。
- 清晰表达开品素材库的真实业务链路：创意图与分析提示词入库，选中后与用户产品图共同参与融合生成。

## 2. 已确认的现状

### 2.1 工作台与模式切换

现有页面已经具备三栏结构。模式切换位于右侧生成输入容器顶部，包含企业、自定义、电商、视频和开品模式。该位置符合“模式只控制当前生成任务”的语义，应保留。

当前主要问题不是结构缺失，而是：

- 面板、标签、输入框和说明文本的尺寸体系不统一。
- 大量内联样式造成边距、圆角、字号和动画规则互相覆盖。
- 左右面板缩放时，部分控件没有 `min-width: 0`、换行或截断规则。
- 下拉菜单、抽屉、遮罩层和聊天内容使用分散的 `z-index`，容易出现边缘冲突。
- hover 放大、上移、弹跳和大范围 `transition: all` 使界面显得粗糙，并可能引起布局抖动。

### 2.2 局部编辑

当前局部编辑链路是真实可用链路：

1. 用户在画布上选择图片并绘制蒙版。
2. 前端生成与原图同为 `1024 × 1024` 的透明蒙版，透明区域表示需要修改，非透明区域表示保护。
3. 前端把原图、蒙版、编辑指令和会话 ID 提交到 `/api/inpaint`。
4. 后端将文件转交给 `GptImageAgent.generateWithMask`，最终调用 `gpt-image-2` 的 `/v1/images/edits`。

现有风险与不足：

- “引用到对话”分别异步准备原图和蒙版，提交时可能出现其中一项尚未准备完成的时序风险。
- 工具仅有画笔、橡皮和套索；撤销只有快捷键，用户无法从界面发现。
- 没有明确展示蒙版是否有效、当前圈选范围是否已准备完成。
- 两条局部编辑提交路径存在重复逻辑，容易出现行为差异。

### 2.3 开品素材库

现有后端链路与目标一致：

1. 用户上传创意素材图。
2. 系统分析素材并生成可编辑提示词。
3. 素材图、标题和分析提示词保存到用户隔离的开品素材数据库。
4. 用户选择自己的白底产品图和一条或多条素材。
5. 每条素材分别以“白底产品图 + 素材图 + 素材提示词”的组合调用生图模型，输出一张融合后的新产品图。

改造重点是让用户在界面上清楚看到“产品主体”和“创意参考”的角色差异，而不是更换后端数据模型。

## 3. 总体信息架构

### 3.1 全局顶部导航

全局导航只保留品牌、文档/帮助、历史与账户等应用级操作。它不承载生成模式切换，避免把“应用导航”和“当前任务设置”混为一层。

高度统一为 52px。按钮使用 32px 图标按钮或紧凑文字按钮，去除圆形大底色和 hover 放大效果。

### 3.2 左侧画布工具区

左侧继续根据画布状态切换内容：

- 未选择图片：显示添加图片入口。
- 单选图片：显示图片信息、裁剪和进入局部编辑。
- 多选图片：显示批量下载、保存和取消选择。
- 局部编辑：显示蒙版工具、大小参数、撤销/重做、清空和提交入口。

左侧面板不常驻展示快捷键说明。快捷键进入按钮的 `title` 或悬浮提示，减少无效占位。

### 3.3 中间无限画布

画布保持主要空间，不改变现有节点拖拽、框选、多选、缩放和批量操作模型。

布局约束：

- 画布容器设置合理最小宽度，左右面板不得将其压缩为不可操作区域。
- resize 时保留当前相机与世界坐标，不因画布像素尺寸重设导致视觉跳动。
- resizer 使用 5px 命中区和 1px 可视线，只有 hover/拖动时显示强调色。
- 空状态、编辑遮罩和画布本体使用固定层级，禁止互相遮挡输入事件。

### 3.4 右侧生成对话区

模式切换固定在生成输入区顶部，采用紧凑分段控件。模式名称允许在窄宽度下缩写或横向滚动，但不能挤压变形。

每种模式继续复用现有配置与提交逻辑，但重新按操作顺序分组：

1. 模式选择。
2. 当前模式的关键参数。
3. 参考图与提示词。
4. 模型选择与主操作按钮。
5. 低频参数放入“高级设置”。

聊天消息、任务进度和生成结果继续位于对话流中。不会新增独立结果页或改变任务接口。

## 4. 视觉系统

### 4.1 色彩

采用中性银灰体系，避免单一蓝色铺满界面：

- 页面背景：冷中性浅灰。
- 工具面板：白色或极浅灰。
- 主文字：深墨灰。
- 次文字：中性灰。
- 主操作：克制的钴蓝色。
- 成功：绿色；警告：琥珀色；危险：红色。
- 深色主题保留，使用石墨灰面板和低饱和蓝色主操作。

禁止使用大面积渐变、装饰光球和无功能的彩色阴影。

### 4.2 字体层级

- 控件标签：12px，常规字重，不使用全大写和额外字距。
- 输入框与按钮：13px。
- 正文与消息：14px。
- 面板标题：14–16px。
- 页面级标题只用于历史、设置和文档等独立页面。

长标签优先换行到两行；仍无法容纳时截断并提供完整悬浮提示。禁止通过继续缩小字号强行塞入。

### 4.3 控件与间距

- 高频输入框、选择器和按钮统一为 34–36px 高。
- 紧凑图标按钮为 30–32px。
- 圆角限定为 4px、6px、8px 三档。
- 间距使用 4、8、12、16、24px 的固定尺度。
- 面板主要依靠背景和 1px 边框分层，只在浮层和拖动元素上使用轻阴影。

### 4.4 动画

动画只表达状态变化：

- hover、选中和焦点：120–150ms。
- 抽屉与模式面板切换：160–180ms。
- 只使用透明度和小距离位移，不使用影响尺寸的缩放、弹跳或 `transition: all`。
- 遵循 `prefers-reduced-motion`，关闭非必要动画。

## 5. 边缘冲突与响应式规则

建立统一层级令牌：

- 基础画布与内容：0–10。
- 固定工具和输入区：20–40。
- 下拉菜单和上下文菜单：100–200。
- 模态框：500。
- 登录与全局阻断层：1000。

所有横向 flex/grid 子项默认设置 `min-width: 0`。标签、模式按钮和工具组必须明确采用换行、滚动或截断中的一种策略。

下拉菜单根据可用空间自动选择向上或向下展开，并限制在视口内；复杂级联选择器继续使用模态框承载，避免被输入容器裁切。

响应式行为：

- 宽屏：左侧工具、中间画布、右侧对话同时显示。
- 中等宽度：先隐藏左侧说明文字并收窄工具栏，再缩小右侧面板。
- 窄屏：左侧工具折叠为图标栏；画布和生成区通过视图切换显示，禁止出现水平页面滚动。

## 6. 局部编辑改造

### 6.1 工具

局部编辑工具扩展为：

- 画笔。
- 橡皮擦。
- 自由套索。
- 矩形选区。
- 撤销。
- 重做。
- 清空蒙版。
- 蒙版显示/隐藏。

保留画笔大小调节。矩形和套索最终都写入同一蒙版画布，不引入新的后端格式。

### 6.2 提交状态

将原图和蒙版准备过程合并为单一 Promise 流程，只有两者都完成后才进入“可提交”状态。

提交前显示：

- 当前编辑图片缩略图。
- 蒙版覆盖区域预览。
- “局部编辑已准备”状态。

如果蒙版为空、原图未准备完成或提示词为空，主按钮保持禁用并给出具体原因。

统一两条重复的局部编辑提交路径，最终只保留一个请求构造函数，确保直接提交和引用到对话后的提交行为一致。

### 6.3 错误处理

- 请求失败时保留原图、蒙版和提示词，允许直接重试。
- 只有生成成功后才清理待提交状态。
- 后端返回非 JSON 时显示可读错误，不出现 `Unexpected token '<'`。
- 生成结果继续进入对话流，并可拖回画布继续编辑。

## 7. 开品素材库交互改造

素材卡片必须同时展示：

- 创意图片。
- 素材标题。
- 分析提示词摘要。
- 选中状态。

用户选中素材后，在开品生成区显示明确的组合预览：

- 左侧：我的产品图，标记为“主体，必须保持功能和识别特征”。
- 右侧：创意素材图，标记为“造型/结构/CMF 创意参考”。
- 下方：本次采用的素材分析提示词，可在本次生成前覆盖编辑，但不强制修改数据库原值。

选择多条素材时，明确显示“将生成 N 个独立方案”，每条素材与产品图分别组成一组请求。生成按钮旁展示预计数量和积分，不把多条素材误解为一次拼贴请求。

后端继续使用现有 `/api/kaipin_material_generate` 和用户隔离数据库，不改变素材存储结构。

## 8. 代码组织

本次不重写整个单页应用，但需要降低继续堆叠内联样式的风险：

- 在 `index.css` 中建立颜色、尺寸、层级和动画令牌。
- 将本次涉及区域的关键内联样式迁移为语义类名。
- 保留现有 DOM ID 和公共函数入口，避免影响已有后端接口与自动化脚本。
- 画布局部编辑的新增状态与工具逻辑优先放入独立脚本模块；如果现有初始化依赖阻止拆分，则先抽取纯函数并保持兼容代理。

## 10. 测试与验收

### 10.1 自动化测试

- 运行完整 Maven 测试。
- 扩展前端 smoke，覆盖模式切换、抽屉展开、下拉菜单、画布初始化和开品素材库打开。
- 为蒙版二值化、透明区语义、空蒙版判断和原图/蒙版异步汇合增加纯函数测试。
- 保留并扩展 `/api/inpaint` 用户上下文测试。

### 10.2 浏览器验证

至少检查以下视口：

- 1440 × 900。
- 1280 × 720。
- 1024 × 768。
- 390 × 844。

每个视口验证：

- 模式按钮、模型选择器和发送按钮不重叠。
- 左右面板缩放时画布不跳动、不出现负宽度。
- 下拉菜单和模态框不被父容器裁切。
- 长标签和动态状态文字不撑破控件。
- 画笔、橡皮、套索、矩形、撤销、重做和清空均可操作。
- 局部编辑请求包含原图、有效透明蒙版和提示词。
- 开品素材卡可选中，产品图与素材图组合关系清晰。

### 10.3 完成标准

- 现有五种模式和生成接口行为保持可用。
- 当前三栏布局与对话流保留。
- 主要边缘冲突、文字挤压和无效动画被消除。
- 局部编辑从“能调用”提升为“状态明确、工具完整、可可靠重试”。
- 开品素材库的图片、分析和融合关系在界面上可直接理解。
