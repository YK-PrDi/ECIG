## 参数传递完整流程

### 1️⃣ 前端参数收集阶段

**文件**：`F:\java\ele-business-java\dist-electron\win-unpacked\resources\frontend\index.html` (行 2968-2991)

用户上传5张图片后，前端通过 **FormData** 构建请求体：

```javascript
// 收集已上传的5张图片
const droppedFiles = [img1, img2, img3, img4, img5];

// 创建 FormData 对象
const formData = new FormData();

// ✅ 关键：每张图片都追加到同一个 'images' 字段
droppedFiles.forEach(f => formData.append('images', f));

// 其他参数
formData.append('prompt', userPrompt);           // 用户提示词
formData.append('count', '1');                   // 生成数量
formData.append('agentId', 'gpt-image');         // 使用的模型
formData.append('sessionId', SESSION_ID);        // 会话ID
formData.append('mode', 'custom');               // 自定义模式标识
formData.append('aspect', '1:1');                // 宽高比
```

**关键点**：
- 5张图片使用**同一个字段名** `'images'` 多次追加
- FormData 支持同名字段多值，Spring 会自动解析成 `List<MultipartFile>`
- 图片以 **File 对象**（二进制）传递，不是 base64

### 4️⃣ 临时文件保存

**位置**：GenerateController.java (行 669-700)

**原因**：MultipartFile 的数据流只在请求周期内有效，必须立即落盘

```java
List<File> whiteTempFiles = new ArrayList<>();
if (hasImages) {
    try {
        // 第1张图作为主参考图 (ref)
        refTempFile = File.createTempFile("ref_", ".jpg");
        images.get(0).transferTo(refTempFile);
        
        // 剩余4张作为白底参考图 (white)
        for (int i = 1; i < images.size(); i++) {
            File wf = File.createTempFile("white_" + i + "_", ".jpg");
            images.get(i).transferTo(wf);
            whiteTempFiles.add(wf);
        }
    } catch (Exception e) {
        return ResponseEntity.internalServerError()
            .body(Map.of("error", "处理上传文件失败: " + e.getMessage()));
    }
}
```

**转换结果**：
- `images` List → 拆分为两部分：
  - `refTempFile`：第1张图（主参考）→ `/tmp/ref_xxx.jpg`
  - `whiteTempFiles`：剩余4张 → `[/tmp/white_1_xxx.jpg, /tmp/white_2_xxx.jpg, ...]`

### 6️⃣ 串行生成逻辑（核心）

**位置**：GenerateController.java (行 783-838)

这是自定义模式的**核心逻辑**，保证系列图片的连贯性：

```java
// 【自定义模式·串行生成】保证系列连贯性
// 策略：第N张图使用「前N-1张已生成的图 + 5张原始参考图」

// 1. 构建基础参考图列表
List<String> baseRefs = new ArrayList<>();
baseRefs.add(refFinal.getAbsolutePath());           // 第1张主参考图
for (File wf : whiteFinal) {
    baseRefs.add(wf.getAbsolutePath());             // 加入4张白底参考图
}
// baseRefs = [ref, white_1, white_2, white_3, white_4]  共5张

// 2. 已生成图片路径列表（动态增长）
List<String> generatedImagePaths = new ArrayList<>();

// 3. 逐张生成
for (String p : promptsFinal) {
    for (int i = 0; i < countFinal; i++) {
        final int imageIndex = i + 1;
        final String outputPath = new File(outputDir, n + ".jpg").getAbsolutePath();

        // ✅ 构建当前图片的参考图列表
        final List<String> currentRefs = new ArrayList<>();
        currentRefs.addAll(generatedImagePaths);    // 已生成的图（第1张生成时为空）
        currentRefs.addAll(baseRefs);               // 5张原始参考图

        // 增强提示词：强调系列连贯性
        final String enhancedPrompt = buildSeriesPrompt(p, imageIndex, countFinal);

        // ✅ 提交生成任务
        CompletableFuture<String> currentFuture = CompletableFuture.supplyAsync(() -> {
            log.info("[自定义模式串行] 开始生成第 {} 张，参考图数量: {}", 
                     imageIndex, currentRefs.size());
            
            boolean ok = imageGenerationService.generateImageMulti(
                    enhancedPrompt,              // 增强的提示词
                    currentRefs,                // 【关键】当前生成需要的所有参考图
                    null,
                    outputPath,
                    generationAgentId,
                    aspectForRefs);
            
            return ok ? outputPath : "失败: 生成未返回图片";
        }, executor);

        // ✅ 立即等待完成（串行关键）
        try {
            String result = currentFuture.get(5, TimeUnit.MINUTES);
            if (!result.startsWith("失败") && new File(result).exists()) {
                generatedImagePaths.add(result);  // 加入已生成列表
                log.info("[自定义模式串行] 第 {} 张生成成功，添加到参考图列表", imageIndex);
            }
        } catch (TimeoutException te) {
            currentFuture.cancel(true);
            log.error("[自定义模式串行] 第 {} 张生成超时", imageIndex);
        }
    }
}
```

**关键机制**：

1. **动态参考图列表**：
   - 第1张生成：`currentRefs` = 5张原始参考图
   - 第2张生成：`currentRefs` = 1张已生成 + 5张原始 = 6张
   - 第3张生成：`currentRefs` = 2张已生成 + 5张原始 = 7张

2. **串行执行**：
   - 每张图提交后立即 `future.get()` 等待完成
   - 完成后才开始生成下一张
   - 保证后续图能"看到"前面的生成结果

3. **提示词增强**：
   - 调用 `buildSeriesPrompt()` 为每张图添加系列连贯性引导
   - 第1张、第2张、第3张的提示词略有不同

### 7️⃣ Service 层参数传递

**文件**：`F:\java\ele-business-java\src\main\java\com\elebusiness\service\ImageGenerationService.java` (行 594-602)

```java
public boolean generateImageMulti(String prompt, 
                                  List<String> refImagePaths,
                                  String whiteBgPath, 
                                  String outputPath,
                                  String agentId, 
                                  String aspect) {
    ImageGeneratorAgent agent = resolveAgent(agentId);
    
    log.info("使用智能体 [{}] 生成图片（refs={}, aspect={}）", 
             agent.getId(),
             refImagePaths == null ? 0 : refImagePaths.size(),
             aspect);
    
    String enforcedPrompt = enforceNoIntersectionPrompt(prompt);
    return agent.generateMulti(enforcedPrompt, refImagePaths, whiteBgPath, outputPath, aspect);
}
```

**参数说明**：
- `refImagePaths`：`List<String>` 包含所有参考图的文件路径
- 路径数量根据生成第几张动态变化（5 → 6 → 7 → ...）

## 核心设计要点

### 1. FormData 同名多值支持
- 前端通过同一字段名 `'images'` 多次 append
- Spring 自动装配成 `List<MultipartFile>`

### 2. 临时文件转换
- MultipartFile 数据流只在请求周期有效
- 必须立即 `transferTo()` 转为本地 File 对象

### 3. 串行生成保证一致性
- 已生成的图作为后续图的参考
- 提示词动态增强（第1/2/3张各有不同引导）
- 每张图等待完成后才生成下一张（`.get()` 阻塞）

### 4. 参考图列表动态增长
- 总是包含原始的 5 张参考图
- 加上已生成的图（0 → 1 → 2 → ...）
- 保证后续图能"看到"前面的生成结果

### 5. COS 上传兜底
- 生成完成后可选上传到云存储
- 或返回本地路径供前端直接访问
