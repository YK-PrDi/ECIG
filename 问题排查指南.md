# 保存图片问题排查指南

## 问题现象
- 你的本机：右键保存图片正常 ✅
- 别人机器：右键保存图片失败，提示保存失败或保存废图片 ❌
- 所有机器：拖拽图片到桌面都正常 ✅

## 根本原因

**环境差异导致路径校验失败**

### 你的开发机：
- `user.dir` = `F:\java\ele-business-java` (项目根目录)
- `tempOutputDir` = `F:\java\ele-business-java\.temp-output`
- 图片路径：`F:\java\ele-business-java\.temp-output\xxx\1.jpg`
- 路径校验：✅ 通过（路径在 tempRoot 下）

### 别人的机器（打包后 exe）：
- `user.dir` = `C:\Users\张三\Desktop` (exe 所在目录)
- `tempOutputDir` = `C:\Users\张三\Desktop\.temp-output` (配置的相对路径)
- 图片实际生成路径：可能不同（取决于打包配置）
- 路径校验：❌ 失败（路径格式不匹配）

---

## 排查步骤

### 1. 查看日志文件

打包后运行 exe，尝试保存图片，然后查看日志：

**日志位置**：
- 控制台输出
- 或应用日志文件（如果配置了）

**关键日志**：
```
INFO  ResourceController - save-to-gallery 请求: tempPath=..., subDir=...
INFO  ResourceController - 路径校验 - tempPath原始: ...
INFO  ResourceController - 路径校验 - tempRoot: ...
INFO  ResourceController - 路径校验 - source标准化: ...
INFO  ResourceController - 路径校验 - source.startsWith(tempRoot): true/false
```

### 2. 确认问题类型

根据日志判断：

#### **情况 A：路径校验失败**
```
ERROR ResourceController - 路径校验失败 - source: C:\Users\张三\..., tempRoot: C:\Users\张三\Desktop\.temp-output
```
**原因**：前端传递的路径与后端 tempRoot 不匹配

**解决方案**：
1. 修改 Electron 启动参数，统一工作目录
2. 或者修改后端校验逻辑（见下文）

#### **情况 B：文件不存在**
```
WARN  ResourceController - 源文件不存在: ...
```
**原因**：临时文件已被 2 小时 TTL 清理

**解决方案**：
1. 延长 TTL 到 24 小时
2. 提示用户尽快保存

#### **情况 C：目录创建失败**
```
ERROR ResourceController - 无法创建目标目录: ...
```
**原因**：磁盘权限或磁盘已满

**解决方案**：
1. 检查磁盘空间
2. 让用户在设置中更改保存位置

---

## 解决方案

### 方案 1：放宽路径校验（推荐）

修改 `ResourceController.java` 的路径校验逻辑：

```java
// 原逻辑：严格校验必须在 tempOutputDir 内
if (!source.startsWith(tempRoot)) {
    return ResponseEntity.badRequest().body(...);
}

// 改为：宽松校验，支持多个可信目录
java.nio.file.Path outputRoot = java.nio.file.Paths.get(
    appProperties.getPaths().getOutputDir()).toAbsolutePath().normalize();

boolean inTempDir = source.startsWith(tempRoot);
boolean inOutputDir = source.startsWith(outputRoot);

if (!inTempDir && !inOutputDir) {
    log.error("路径校验失败 - 不在可信目录内");
    return ResponseEntity.badRequest().body(Map.of("success", false,
        "error", "非法路径：文件必须在应用目录内"));
}
```

### 方案 2：统一工作目录（Electron 配置）

在 Electron 启动 Java 时，显式设置工作目录：

```javascript
// main.js 或启动脚本
const appDataPath = path.join(app.getPath('userData'), 'app-data');
fs.mkdirSync(appDataPath, { recursive: true });

const javaProcess = spawn('java', [
  '-jar',
  'app.jar',
  `-Duser.dir=${appDataPath}`,  // 强制设置工作目录
  `-Dapp.packaged=true`
], {
  cwd: appDataPath  // 子进程工作目录
});
```

### 方案 3：使用绝对路径配置

修改 `application.yml`，打包时注入绝对路径：

```yaml
app:
  paths:
    # 开发态用相对路径，打包态用绝对路径
    temp-output-dir: "${app.data.dir:./}/.temp-output"
    output-dir: "${app.data.dir:./}/生成结果"
```

然后在 Electron 启动时注入：
```javascript
-Dapp.data.dir=C:/Users/用户名/AppData/Local/YourApp
```

## 快速诊断命令

让别人运行 exe 后，执行以下操作获取诊断信息：

1. **生成一张图片**
2. **打开浏览器开发者工具**（F12）
3. **在 Console 输入**：
```javascript
// 获取路径映射
console.log('pathMap:', window.__pathMap);

// 获取配置
fetch('/api/settings').then(r => r.json()).then(d => console.log('settings:', d));
```

4. **尝试保存，查看网络请求**
   - Network 标签 → 找到 `/api/save-to-gallery` 请求
   - 查看 Request Payload: `{ tempPath: "..." }`
   - 查看 Response: `{ success: false, error: "..." }`

5. **将以上信息发给你进行诊断**

## 临时绕过方案

如果用户急需保存图片：

1. **使用拖拽功能** — 将图片拖到桌面（这个功能正常）
2. **手动进入临时目录** — 打开 `exe所在目录\.temp-output\`，手动复制图片
3. **启用 COS 云存储** — 配置后图片会上传到云端，不受本地路径限制
