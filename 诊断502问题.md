# 502 错误诊断指南

## 问题描述
前端点击保存按钮时报错：`Failed to load resource: the server responded with a status of 502 ()`

## 诊断步骤

### 1. 确认部署架构
**问题：前端是如何访问后端的？**
- [ ] Electron 应用（打包的桌面应用）
- [ ] 浏览器直接访问（如 http://服务器IP:5020）
- [ ] 通过 Nginx/Apache 反向代理

### 2. 检查后端日志
**位置：** `数据目录/logs/app.log`

**查找关键字：**
```bash
grep "save-to-gallery" logs/app.log
grep "路径校验" logs/app.log
grep "ERROR" logs/app.log | tail -50
```

**预期看到：**
```
INFO  c.e.controller.ResourceController - save-to-gallery 请求: tempPath=xxx, subDir=xxx
INFO  c.e.controller.ResourceController - 路径校验 - tempPath原始: xxx
INFO  c.e.controller.ResourceController - 解析输出目录: xxx
```

### 3. 检查是否有拦截器/网关
**如果使用 Nginx，检查配置：**
```nginx
# 检查是否有 proxy_read_timeout 配置
proxy_read_timeout 60s;  # 太短会导致 502

# 检查是否有 client_max_body_size 限制
client_max_body_size 100M;

# 检查 upstream 配置
upstream backend {
    server localhost:5020 max_fails=3 fail_timeout=30s;
}
```

### 4. 测试接口是否正常
**直接测试后端：**
```bash
# 测试保存接口（需要先登录）
curl -X POST http://localhost:5020/api/save-to-gallery \
  -H "Content-Type: application/json" \
  -H "Cookie: JSESSIONID=你的session" \
  -d '{"tempPath":"测试路径"}' \
  -v
```

### 5. 常见原因

#### 原因1：请求超时
- **症状：** 保存大文件时出现，小文件正常
- **解决：** 增加 Nginx `proxy_read_timeout` 或 Tomcat `connection-timeout`

#### 原因2：后端服务崩溃
- **症状：** 所有接口都返回 502
- **解决：** 检查 Java 进程是否存在，查看 logs/jvm.log

#### 原因3：路径校验失败
- **症状：** 后端返回 400 或 500，但前端看到 502
- **解决：** 检查日志中的"路径校验失败"错误

#### 原因4：数据库锁
- **症状：** 第一次保存正常，后续保存卡住
- **解决：** 检查 SQLite 数据库是否被锁，重启服务

#### 原因5：磁盘空间不足
- **症状：** 文件复制失败
- **解决：** 检查磁盘空间：`df -h`

#### 原因6：Session 过期
- **症状：** 返回 401，但前端误报 502
- **解决：** 重新登录

## 下一步

### 如果日志中没有 save-to-gallery 请求记录
→ **说明请求根本没到达 Spring Boot**
- 检查网络连接
- 检查反向代理配置
- 检查防火墙规则

### 如果日志中有请求记录但卡在某一步
→ **根据最后一条日志定位**
- "路径校验" → 检查路径是否正确
- "文件复制" → 检查磁盘 I/O 和权限
- "markSaved" → 检查数据库是否正常

### 如果日志中显示请求成功
→ **问题在前端或中间层**
- 检查前端是否正确解析响应
- 检查浏览器控制台的网络请求详情
- 检查是否有 CORS 问题

## 临时解决方案

### 方案1：禁用路径校验（不推荐）
如果是路径校验导致的问题，可以临时注释掉 310-314 行的校验代码

### 方案2：增加超时时间
在 application.yml 中增加：
```yaml
server:
  tomcat:
    connection-timeout: 120000  # 改为 2 分钟
```

### 方案3：改为异步保存（需要修改代码）
将保存操作改为异步，立即返回任务ID，前端轮询任务状态

5. 问题是否稳定复现（每次都502还是偶尔）
