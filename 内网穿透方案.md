# 内网穿透方案（让本地 GPU 可被远程访问）

## ⚠️ 警告
- 这种方案复杂，不推荐新手使用
- 网络不稳定，可能断连
- 安全风险较高（暴露本地服务）

---

## 方案 1: frp（免费，需要有公网服务器）

### 1. 准备
```
需要:
- 一台有公网 IP 的服务器（frp 服务端）
- 你的本地电脑（frp 客户端 + SD WebUI）
```

### 2. 服务器端配置

```bash
# 下载 frp
wget https://github.com/fatedier/frp/releases/download/v0.52.0/frp_0.52.0_linux_amd64.tar.gz
tar -xzf frp_0.52.0_linux_amd64.tar.gz
cd frp_0.52.0_linux_amd64

# 编辑 frps.toml
cat > frps.toml << EOF
bindPort = 7000
EOF

# 启动服务端
./frps -c frps.toml
```

### 3. 本地客户端配置

```powershell
# 下载 frp Windows 版
# https://github.com/fatedier/frp/releases/download/v0.52.0/frp_0.52.0_windows_amd64.zip

# 解压后编辑 frpc.toml
[common]
server_addr = "你的服务器IP"
server_port = 7000

[[proxies]]
name = "sd-webui"
type = "tcp"
local_ip = "127.0.0.1"
local_port = 7860
remote_port = 7860

# 启动客户端
frpc.exe -c frpc.toml
```

### 4. 修改 Java 配置

```yaml
# 部署在服务器的 Java 项目
app:
  local-sd:
    api-url: "http://localhost:7860"  # 通过 frp 转发到你的本地
```

## 方案 2: ngrok（最简单，但免费版限速）

### 1. 注册 ngrok
```
官网: https://ngrok.com/
注册并获取 authtoken
```

### 2. 启动 SD WebUI
```powershell
D:\stable-diffusion-webui\webui-user.bat
```

### 3. 启动 ngrok 穿透
```powershell
# 安装 ngrok
# 下载: https://ngrok.com/download

# 配置 authtoken
ngrok config add-authtoken <你的token>

# 穿透 7860 端口
ngrok http 7860

# 会得到一个公网 URL:
# Forwarding: https://xxxx-xx-xx-xx-xx.ngrok-free.app -> http://localhost:7860
```

### 4. 修改 Java 配置
```yaml
app:
  local-sd:
    api-url: "https://xxxx-xx-xx-xx-xx.ngrok-free.app"
```

## 方案 3: ZeroTier（虚拟局域网）

### 1. 注册 ZeroTier
```
官网: https://www.zerotier.com/
创建网络，获取 Network ID
```

### 2. 本地电脑安装
```powershell
# 下载: https://www.zerotier.com/download/
# 安装后加入网络
zerotier-cli join <Network ID>
```

### 3. 服务器安装
```bash
curl -s https://install.zerotier.com | sudo bash
sudo zerotier-cli join <Network ID>
```

### 4. 修改 Java 配置
```yaml
# ZeroTier 分配的虚拟 IP（如 10.147.20.x）
app:
  local-sd:
    api-url: "http://10.147.20.123:7860"
```

## ✅ 推荐决策

**如果只是自己用**:
→ 全部本地运行（场景 A），不需要内网穿透

**如果要给团队用**:
→ 租一台带 GPU 的云服务器，部署 SD WebUI + Java

**如果必须用本地 GPU 且要远程访问**:
→ 用 ngrok（最简单）或 frp（最稳定）

---

## 🎯 我的建议

1. **先本地测试**（场景 A）
   - 验证功能是否正常
   - 测试生成效果
   - 不花钱，最简单

2. **确认效果后再决定**
   - 如果只是偶尔用 → 继续本地
   - 如果要给别人用 → 部署到云服务器
   - 如果要远程用本地 GPU → 用 ngrok
