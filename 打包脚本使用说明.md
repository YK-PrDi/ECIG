# 一键打包脚本使用说明

## 🚀 快速开始

### 方式 1：智能打包（推荐）

```bash
auto-build.bat
```

**自动完成：**
- ✅ 自动检测 Maven（便携版 → 系统 PATH → MAVEN_HOME）
- ✅ 自动关闭运行中的 AI Studio
- ✅ 自动清理旧文件
- ✅ Maven 编译
- ✅ Electron 打包

**完全清理模式**（包括 JRE runtime）：
```bash
auto-build.bat --clean
```

---

### 方式 2：快速打包（需手动配置 Maven 路径）

```bash
quick-build.bat
```

**需要修改**：编辑脚本第 36 行，设置你的 Maven 路径：
```batch
set "MAVEN_HOME=C:\Users\你的用户名\maven-portable\apache-maven-3.9.9"
```

---

### 方式 3：原始打包脚本

```bash
build-dist.bat --no-bump
```

**前提条件**：
- Maven 必须在系统 PATH
- 手动关闭运行中的 AI Studio

## 🐛 问题排查

### 问题 1：打包失败 - "Maven 未找到"

**解决方案 A**（推荐）：使用 `auto-build.bat`，会自动检测

**解决方案 B**：手动配置
1. 下载 Maven 便携版到 `%USERPROFILE%\maven-portable\`
2. 或安装 Maven 并添加到系统 PATH

---

### 问题 2：打包失败 - "文件被占用"

**原因**：AI Studio 还在运行

**解决方案**：
1. 使用 `auto-build.bat`（会自动关闭）
2. 或手动打开任务管理器，结束所有 `AI Studio.exe` 进程

---

### 问题 3：Electron 打包慢

**正常现象**：首次打包需要 5-10 分钟

**加速方法**：
- 第二次打包会快很多（复用 runtime）
- 使用 `auto-build.bat` 而非 `--clean` 模式

---

## 📝 诊断别人机器的问题

### 步骤 1：分发新版本
- 安装版：`AI Studio Setup 1.2.3.exe`
- 便携版：`win-unpacked/AI Studio.exe`

### 步骤 2：让对方复现问题
1. 启动应用
2. 生成图片
3. 右键保存（如果失败）

### 步骤 3：获取日志
让对方：
1. 打开他们选择的数据目录
2. 找到 `logs/` 文件夹
3. 发送 `app.log`（和 `jvm.log` 如果启动失败）

### 步骤 4：诊断问题
根据日志内容，对照《问题排查指南.md》：

| 日志特征 | 问题 | 解决方案 |
|---------|------|---------|
| `可写=false` | 目录权限问题 | 重新选择数据目录 |
| `startsWith(tempRoot): false` | 路径校验失败 | 放宽校验逻辑 |
| `源文件不存在` | 临时文件被清理 | 延长 TTL 或提示用户 |
